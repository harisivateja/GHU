Data Design & Model

Synthetic dataset architecture and generation methodology

Dataset Overview: The Centralized Student Dashboard prototype utilizes a synthetic dataset comprising 1,000 student records distributed across six normalized entities. Created programmatically using Python (pandas and faker libraries), the dataset simulates realistic university data with intentional correlations to test the predictive risk model.

1,000 Student Records
6 Data Entities
2.77 Average GPA
3NF Normalization Level

Data Entities

Students

Primary Entity: Core student demographic and academic information

  • Student_ID (Primary Key)
  • First_Name, Last_Name
  • Department
  • GPA (0.0-4.0)
  • Attendance_Rate (%)
  • Credits_Completed
  • Engagement_Score (0-100)

Financials

Financial Data: Tuition, payments, and scholarship status

  • Finance_ID (Primary Key)
  • Student_ID (Foreign Key)
  • Tuition_Fees
  • Amount_Paid
  • Outstanding_Balance
  • Scholarship_Status (0/1)

Housing

Residence Data: On-campus vs off-campus housing

  • Housing_ID (Primary Key)
  • Student_ID (Foreign Key)
  • Housing_Status
  • Building_Name
  • Room_Number

Enrollments

Course Registration: Student-course relationships

  • Enrollment_ID (Primary Key)
  • Student_ID (Foreign Key)
  • Course_ID (Foreign Key)
  • Term
  • Grade
  • Attendance_Percentage

Courses

Course Catalog: Course details and credit information

  • Course_ID (Primary Key)
  • Course_Name
  • Department
  • Credits

Alerts & Advisors

Tracking System: Student alerts and advisor assignments

  • Alert_ID (Primary Key)
  • Student_ID (Foreign Key)
  • Advisor_ID (Foreign Key)
  • Alert_Type
  • Alert_Date
  • Description

Dataset Generation Methodology

The synthetic data was created with logical coherence and referential integrity to simulate real-world university data:

  • Referential Integrity: Student_ID serves as the primary Foreign Key, explicitly linking records across all entities (Financials, Housing, Enrollments, Alerts) back to the Students entity.
  • Realistic Distributions: GPA values randomly generated within 0.0-4.0 range with strategic skew towards institutional average of 2.77; attendance rates and engagement scores follow bell curve distributions.
  • Risk Simulation: Intentional positive correlation introduced between risk factorsâ€”students with lower GPA (â‰¤2.5) had higher probability of large Outstanding Balance or low Attendance Rate.
  • Normalization (3NF): All six entities normalized to Third Normal Form to remove redundancy; course details stored once in Courses entity, with Enrollments storing only Course_ID as Foreign Key.
  • Data Volume: 1,000 student records with proportional enrollment records (~3-5 courses per student), ensuring sufficient data for meaningful analytics.
  • Python Libraries: Generated programmatically using pandas for data manipulation and faker for realistic name/text generation.
